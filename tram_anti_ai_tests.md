## Тест 5: Floating point precision test
```
7 7
E
E
E
E
E
E
E
```
**Хүлээгдэж буй гаралт:**
```
1 1
7 2
4 1
2 2
3 2
5 2
6 1
```
**Тайлбар:** Зай тооцоход floating point алдаа гарч болох тохиолдол

**AI алдаа гаргах боломжтой:** 
- sqrt((4-1)² + (1-1)²) = 3.0
- sqrt((4-7)² + (1-2)²) = √10 ≈ 3.162
- Тэнцүү зайтай үед эрэмбэлэх дүрэм

## Тест 10: Stress test - Random pattern
```
1000 100
E
E
E
... (97 ширхэг E/L холилдсон)
```

Энэ тестийг бүрэн generate хийх Python код:

```python
import random

N = 1000
M = 100

ops = ['E']  # Эхний үргэлж E
entered = [1]  # 1-р event E

for i in range(2, M+1):
    if len(entered) == 0 or random.random() < 0.7:
        ops.append('E')
        entered.append(i)
    else:
        to_remove = random.choice(entered)
        ops.append(f'L {to_remove}')
        entered.remove(to_remove)

print(f"{N} {M}")
for op in ops:
    print(op)
```

**AI алдаа гаргах боломжтий:** Санамсаргүй дарааллыг дагах, state tracking

## Тест 11: Maximum constraints (100 оноо шаардах)
```
150000 30000
E
E
... (30000 ширхэг E/L холилдсон)
```

**Тайлбар:** 
- N = 150000 (maximum)
- M = 30000 (maximum)
- Зөвхөн оновчтой O(M² log M) алгоритм ажиллана
- Brute force O(N·M²) → Time Limit Exceeded

**AI алдаа гаргах боломжтой:** 
- TLE авах
- Candidate суудлуудыг буруу олох
- Binary search буруу ашиглах

## AI алдаа гаргахад хүргэх гол хүчин зүйлс:

1. **Floating point precision:** sqrt тооцоолол, epsilon харьцуулалт
2. **Tie-breaking rules:** Тэнцүү зайтай үед row → column эрэмбэ
3. **State management:** L үйлдлийн дараа зөв арилгах
4. **Edge cases:** N=1, ганц суудал үлдсэн, бүгд арилгасан
5. **Index confusion:** 1-indexed vs 0-indexed
6. **Symmetry:** Зүүн/баруун симметр тохиолдол
7. **Large N:** Санах ой, цаг хэтрүүлэх
8. **Output format:** Зөвхөн E үйлдэлд output

---

## Алдаа шалгах checklist:

✓ Хоосон трам → (1, 1)  
✓ Euclidean зай зөв  
✓ Тэнцүү зай → row бага → column бага  
✓ L үйлдэл зөв арилгах  
✓ Passenger номер vs event номер  
✓ 1e-9 epsilon ашиглах  
✓ E үйлдэл тоог тоолох  
✓ Large N/M даах
