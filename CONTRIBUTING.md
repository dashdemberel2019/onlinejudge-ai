## How to Contribute

### 1. Reporting Issues

**Found an error?**
- Check existing issues first
- Open a new issue with:
  - Clear description
  - Document/section reference
  - Expected vs actual information
  - Sources (if correcting data)

**Categories:**
- `bug` - Factual errors
- `typo` - Spelling/grammar
- `outdated` - Information needs update
- `enhancement` - Suggestions for improvement

---

### 2. Suggesting Improvements

**Have an idea?**
- Open an issue with `enhancement` label
- Describe the improvement
- Explain why it's useful
- Provide examples if applicable

**Examples:**
- Additional research topics
- New detection methods
- Case studies
- Translation to other languages

---

### 3. Submitting Changes

**Process:**

1. **Fork the repository**
   ```bash
   # Click "Fork" button on GitHub
   ```

2. **Clone your fork**
   ```bash
   git clone https://github.com/YOUR_USERNAME/competitive-programming-research.git
   cd competitive-programming-research
   ```

3. **Create a branch**
   ```bash
   git checkout -b feature/your-improvement
   # or
   git checkout -b fix/issue-number
   ```

4. **Make changes**
   - Edit markdown files
   - Add sources/citations
   - Follow style guide (below)

5. **Commit changes**
   ```bash
   git add .
   git commit -m "Add: description of changes"
   ```

6. **Push to your fork**
   ```bash
   git push origin feature/your-improvement
   ```

7. **Open Pull Request**
   - Go to original repository
   - Click "New Pull Request"
   - Select your branch
   - Fill in description

## Style Guide

### Markdown Formatting

**Headers:**
```markdown
# Top Level (Document Title)
## Major Section
### Subsection
#### Detail Level
```

**Lists:**
```markdown
- Unordered item
  - Nested item
  
1. Ordered item
2. Another item
```

**Code blocks:**
````markdown

    pass

```

```
````

**Tables:**
```markdown
| Column 1 | Column 2 |
|----------|----------|
| Value A  | Value B  |
```
